# Git Conventions — LONO

## Commit message формат
```
[тип]: короткий опис (українською або англійською)
```

## Типи
- `feat` — нова сторінка або секція
- `fix` — виправлення верстки або контенту
- `content` — зміни в Stories/*.md файлах
- `style` — зміни CSS без впливу на функціональність
- `refactor` — покращення коду без зміни поведінки
- `docs` — оновлення документації (CLAUDE.md та ін.)
- `chore` — налаштування, конфіги

## Приклади
```
feat: додати секцію команди лікарів на головну
fix: виправити адаптивність hero-секції на 375px
content: оновити опис пакетів у Stories/01-pakety
style: уніфікувати кольорові змінні CSS
```

## Заборонено комітити
- Файли з паролями, API-ключами, секретами
- Великі бінарні файли без обговорення (відео, архіви)
- Тимчасові файли (`*.tmp`, `*.log`, `.DS_Store`)
