# Security Review Skill — LONO

Автоматично викликається коли: змінюються файли з формами, зовнішніми запитами, або будь-які скрипти.

## Що перевіряти

### XSS (Cross-site Scripting)
- Чи не використовується `innerHTML` з неперевіреними даними
- Чи екрануються дані перед вставкою в DOM
- Чи використовується `textContent` замість `innerHTML` де можливо

### Форми та дані
- Валідація на рівні HTML (`required`, `type`, `pattern`)
- Чи не відправляються форми на зовнішні сервіси без HTTPS
- Чи не логуються у консоль персональні дані пацієнтів

### Зовнішні ресурси
- HTTPS для всіх зовнішніх посилань та шрифтів (Google Fonts тощо)
- Чи перевірені CDN-бібліотеки (якщо є)
- `rel="noopener noreferrer"` для зовнішніх посилань з `target="_blank"`

### Конфіденційність
- Жодних hardcoded API-ключів, токенів, паролів у коді
- Форма запису — дані пацієнтів не зберігаються на клієнті

## Формат звіту
Повертай список знайдених проблем із рівнями:
- 🔴 КРИТИЧНО — виправити до публікації
- 🟡 ПОПЕРЕДЖЕННЯ — виправити бажано
- 🟢 ОК — проблем не виявлено
